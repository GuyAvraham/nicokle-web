Prifender is crawling the corporate network, automatically discover repositories containing PII and then associates key data elements with each of the identities. Some of the data elements tag the obligations that are associated with each of the identities. Once the inventory of identities with their respective obligations is created, meta data (e.g repository location, data access time stamp etc.) is added and attached as an attribute of each identity
